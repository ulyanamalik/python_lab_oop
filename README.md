# Лабораторная работа №1:Транспорт (5 вариант)

## Цель работы

- Освоить объявление пользовательских классов
- Разобраться с инкапсуляцией (закрытые поля)
- Реализовать свойства (`@property`)
- Переопределить магические методы (`__str__`, `__repr__`, `__eq__`)
- Понять разницу между атрибутами класса и экземпляра

  
## Основные идеи:

Инкапсуляция – данные объекта (номер маршрута, вместимость, количество пассажиров, состояние) скрыты за защищёнными атрибутами (с одним подчёркиванием), а доступ к ним организован через свойства (@property) с методами-валидаторами.

Проверка корректности данных – при создании объекта и изменении его атрибутов выполняются проверки типов и логических ограничений (например, вместимость > 0, пассажиры не могут быть отрицательными и не превышать вместимость, статус только из допустимого набора). Валидация вынесена в отдельные методы, что улучшает читаемость и повторное использование.

Атрибуты класса – продемонстрировано использование атрибутов, общих для всех экземпляров (счётчик созданных автобусов total_buses и кортеж допустимых статусов VALID_STATUSES).

Магические методы – переопределены __str__ (для пользовательского вывода), __repr__ (для отладки) и __eq__ (для сравнения объектов по номеру маршрута).

Бизнес-логика, зависящая от состояния – методы board, alight, start_route, park реализуют поведение, которое меняется в зависимости от текущего статуса автобуса (например, посадка разрешена только на маршруте, парковка – только без пассажиров).

Обработка исключений – при нарушении условий валидации или бизнес-правил выбрасываются исключения ValueError, которые перехватываются в демонстрационном скрипте, показывая корректную реакцию программы на недопустимые действия.

Таким образом, код наглядно иллюстрирует, как можно создавать надёжные, самопроверяемые классы с понятным интерфейсом и внутренней логикой.

## Реализованный класс Bus

Представляет автобус с номером маршрута, вместимостью, текущим количеством пассажиров и состоянием.


**Атрибуты класса**

`total_buses` – целое число, счётчик созданных экземпляров класса. Увеличивается при каждом вызове конструктора.

`VALID_STATUSES` – кортеж строк ('parked', 'on_route'), задающий допустимые состояния автобуса. Используется для валидации статуса.

**Атрибуты экземпляра (защищённые)**

`_route` – строка, номер маршрута.

`_capacity` – целое положительное число, максимальная вместимость.

`_passengers` – целое неотрицательное число, текущее количество пассажиров.

`_status` – строка, состояние автобуса (одно из VALID_STATUSES).

**Методы валидации (приватные)**

`_validate_route(value)` – проверяет, что значение является непустой строкой.

`_validate_capacity(value)` – проверяет, что значение – целое положительное число.

`_validate_passengers(value, capacity=None)` – проверяет, что пассажиры – целое ≥ 0 и (если передан capacity) не превышает вместимость.

`_validate_status(value)` – проверяет, что статус входит в VALID_STATUSES.

**Свойства (@property)**

`route` – геттер для номера маршрута (только чтение).

`capacity` – геттер и сеттер с вызовом _validate_capacity.

`passengers` – геттер и сеттер с вызовом _validate_passengers.

`status` – геттер и сеттер с вызовом _validate_status.

**Бизнес-методы**

`board(num)` – посадка num пассажиров. Доступна только при статусе 'on_route'. Проверяет, что num > 0 и что после посадки не превышается вместимость.

`alight(num)` – высадка num пассажиров. Проверяет num > 0 и наличие достаточного числа пассажиров.

`start_route()` – переводит автобус из состояния 'parked' в 'on_route'. Если статус не 'parked', генерирует исключение.

`park()` – возвращает автобус с маршрута в парк. Требует статуса 'on_route' и отсутствия пассажиров (passengers == 0).

**Магические методы**

`__str__()` – возвращает строку вида: "Автобус {route}: {passengers}/{capacity} пасс., статус: {status}".

`__repr__()` – возвращает строку, которая могла бы быть использована для воссоздания объекта: "Bus('{route}', {capacity}, {passengers}, '{status}')".

`__eq__(other)` – сравнивает текущий объект с other по номеру маршрута. Если other не является экземпляром Bus, возвращает NotImplemented.
