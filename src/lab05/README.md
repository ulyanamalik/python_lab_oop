# Лабораторная работа №5: Функции как аргументы. Стратегии и делегаты

## Цель работы

Освоить передачу функций как аргументов в другие функции и методы, научиться применять встроенные функции высшего порядка (`map`, `filter`, `sorted`), понять концепцию паттерна «Стратегия» и реализовать его на Python, освоить `lambda`-выражения и интегрировать функциональный стиль с объектно-ориентированным кодом из предыдущих лабораторных работ.

## Реализованные функции и стратегии

### 1. Функции-стратегии для сортировки (файл `strategies.py`)

| Функция | Описание |
|---------|----------|
| `by_route(bus)` | Ключ для сортировки по номеру маршрута |
| `by_capacity(bus)` | Ключ для сортировки по вместимости |
| `by_passengers(bus)` | Ключ для сортировки по количеству пассажиров |
| `by_route_then_capacity(bus)` | Сортировка сначала по маршруту, затем по вместимости (кортеж) |

### 2. Функции-фильтры

| Функция | Описание |
|---------|----------|
| `is_on_route(bus)` | Отбирает автобусы на маршруте |
| `is_parked(bus)` | Отбирает автобусы в парке |
| `has_passengers(bus)` | Отбирает автобусы с пассажирами |
| `is_city_bus(bus)` | Отбирает городские автобусы |
| `is_tourist_bus(bus)` | Отбирает туристические автобусы |

### 3. Фабрики функций

| Фабрика | Описание |
|---------|----------|
| `capacity_less_than(limit)` | Создаёт фильтр для отбора автобусов с вместимостью < limit |
| `capacity_greater_than(limit)` | Создаёт фильтр для отбора автобусов с вместимостью > limit |
| `apply_discount(percent)` | Создаёт функцию для применения скидки (демонстрация `map`) |

### 4. Callable-объекты (паттерн «Стратегия»)

| Класс | Описание |
|-------|----------|
| `RouteSorter` | Стратегия сортировки по маршруту |
| `CapacitySorter` | Стратегия сортировки по вместимости |
| `PassengerSorter` | Стратегия сортировки по пассажирам |
| `OnRouteFilter` | Стратегия фильтрации автобусов на маршруте |
| `ParkedFilter` | Стратегия фильтрации автобусов в парке |
| `AddPassengersStrategy` | Стратегия добавления пассажиров (обработка) |

### 5. Новые методы коллекции `BusFleet` (файл `collection.py`)

| Метод | Описание |
|-------|----------|
| `sort_by(key_func, reverse=False)` | Сортирует коллекцию по переданной функции-ключу |
| `filter_by(predicate)` | Возвращает новую коллекцию с отфильтрованными элементами |
| `apply(func)` | Применяет функцию ко всем элементам, возвращает список результатов |
| `map_to(transform_func)` | Преобразует коллекцию в список через `map` |

## Демонстрация работы (сценарии из `demo.py`)

### Сценарий 1: Сортировка с разными стратегиями
Создаётся коллекция из 6 автобусов (обычные, городские, туристические). 

К коллекции применяются разные стратегии сортировки: по маршруту (by_route), по вместимости (by_capacity), по количеству пассажиров (by_passengers) и составная сортировка (сначала по маршруту, затем по вместимости).

 Каждый раз выводится коллекция, отсортированная по разному принципу. Это демонстрирует гибкость подхода — одна и та же коллекция может быть отсортирована разными способами без изменения её кода.

### Сценарий 2: Фильтрация с разными функциями
Применяются две функции-фильтра: is_on_route (отбирает автобусы на маршруте) и has_passengers (отбирает автобусы, в которых есть пассажиры). 

Функция filter() применяет каждую из них к коллекции. 

Выводятся списки автобусов, удовлетворяющих каждому условию. Это показывает, как можно легко отбирать элементы коллекции по разным критериям.

### Сценарий 3: Применение map() для преобразования
Демонстрируются три способа преобразования коллекции через map():

 извлечение номеров маршрутов (to_route), 
 
 преобразование автобусов в строки (to_string) и применение скидки через фабрику функций apply_discount().

 В последнем случае создаётся функция скидки с параметром 10%, которая затем применяется ко всем автобусам. 
 
 Это показывает гибкость map() для массового преобразования данных.

### Сценарий 4: Фабрика функций для фильтрации
Создаются две фабрики функций: capacity_less_than(limit) и capacity_greater_than(limit). 

Они возвращают функции-фильтры, которые отбирают автобусы с вместимостью меньше 40 и больше 45 соответственно. 

Фабрики позволяют создавать фильтры с разными пороговыми значениями без дублирования кода. 

Выводятся списки отобранных автобусов.

### Сценарий 5: Методы коллекции sort_by() и filter_by()
Демонстрируются новые методы, добавленные в коллекцию BusFleet: sort_by(by_capacity) сортирует коллекцию по вместимости, а filter_by(is_parked) возвращает новую коллекцию из автобусов в парке. 

В отличие от встроенных sorted() и filter(), эти методы работают напрямую с коллекцией и поддерживают цепочки вызовов.

### Сценарий 6: Сравнение lambda и именованной функции

Выполняется сортировка коллекции двумя способами: через lambda-выражение (key=lambda b: b.route) и через именованную функцию (key=by_route). 

Результаты сортировки выводятся и сравниваются. 

Показывается, что оба подхода дают одинаковый результат.

 Lambda удобна для простых одноразовых операций, а именованная функция — когда логика переиспользуется в нескольких местах.

### Сценарий 7: Callable-объекты (паттерн «Стратегия»)

Создаются callable-объекты — классы с методом call: RouteSorter для сортировки по маршруту и OnRouteFilter для фильтрации автобусов на маршруте.

 Эти объекты передаются в sorted() и filter() как обычные функции. 
 
 Callable-объекты могут хранить внутреннее состояние, что даёт больше возможностей, чем обычные функции. 
 
 Это демонстрирует реализацию паттерна «Стратегия» через объекты.


### Сценарий 8: Метод apply() — применение функции ко всем элементам
Создаётся callable-объект AddPassengersStrategy(5), который добавляет 5 пассажиров к каждому автобусу. 

Метод apply() применяет эту стратегию ко всем элементам коллекции и возвращает список результатов. 

Если автобус на маршруте — пассажиры добавляются успешно, если в парке — выводится сообщение об ошибке. 

Это показывает, как можно массово обрабатывать все объекты коллекции одной стратегией.

### Сценарий 9: Цепочка операций (filter → sort → map)
Демонстрируется последовательное применение нескольких операций к коллекции в одной строке: сначала фильтрация (filter_by(is_on_route)), затем сортировка (sort_by(by_capacity)), затем преобразование (map_to(to_route)). 

Каждый метод возвращает результат, который передаётся следующему.

 Это делает код читаемым, декларативным и позволяет записывать сложные преобразования компактно.

### Сценарий 10: Замена стратегии без изменения кода коллекции

Показывается, что коллекция может использовать разные стратегии сортировки без изменения своего кода.

 Метод sort_by() вызывается трижды с разными функциями-ключами: by_route (сортировка по маршруту), by_capacity (по вместимости) и by_passengers (по пассажирам).
 
Выводится порядок маршрутов после каждой сортировки. Код коллекции остаётся неизменным — меняется только переданная стратегия. Это и есть суть паттерна «Стратегия».

### Сценарий 11: Фильтрация по типу (городские/туристические)
Используются функции is_city_bus и is_tourist_bus, которые внутри применяют isinstance() для проверки типа объекта. 

Метод filter_by() отбирает только городские или только туристические автобусы.

 Выводятся списки маршрутов отобранных автобусов.
 
  Это показывает, как можно фильтровать коллекцию не только по значениям атрибутов, но и по типу объектов, что полезно для операций над подмножеством коллекции.
