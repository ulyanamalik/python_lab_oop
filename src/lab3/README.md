## Описание реализованной иерархии классов

### Базовый класс `Bus` (файл `base.py`)

Класс был реализован в лабораторной работе №1 и №2. Он описывает обычный автобус.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `route` | str | Номер маршрута |
| `capacity` | int | Вместимость автобуса |
| `passengers` | int | Текущее количество пассажиров |
| `status` | str | Состояние (`parked` или `on_route`) |

**Основные методы:**
- `board(num)` – посадка пассажиров
- `alight(num)` – высадка пассажиров
- `start_route()` – выезд на маршрут
- `park()` – возврат в парк
- `get_bus_type()` – возвращает тип автобуса (переопределяется)
- `calculate_price(distance_km)` – рассчитывает стоимость поездки (переопределяется)

### Дочерний класс `CityBus` (файл `models.py`)

Наследуется от `Bus`. Добавляет специфику городских автобусов.

| Новый атрибут | Тип | Описание |
|---------------|-----|----------|
| `route_type` | str | Тип маршрута (например, "городской", "пригородный") |
| `has_air_conditioning` | bool | Наличие кондиционера |

**Новые методы:**
- `get_route_type()` – возвращает тип маршрута
- `has_ac()` – возвращает наличие кондиционера

**Переопределённые методы:**
- `get_bus_type()` → возвращает `"Городской автобус"`
- `calculate_price()` → `distance_km * 15` (дешевле обычного)
- `__str__()` – добавляет информацию о типе маршрута и кондиционере

### Дочерний класс `TouristBus` (файл `models.py`)

Наследуется от `Bus`. Добавляет специфику туристических автобусов.

| Новый атрибут | Тип | Описание |
|---------------|-----|----------|
| `has_toilet` | bool | Наличие туалета |
| `comfort_level` | int | Уровень комфорта (1–5) |

**Новые методы:**
- `has_toilet()` – возвращает наличие туалета
- `get_comfort_level()` – возвращает уровень комфорта

**Переопределённые методы:**
- `get_bus_type()` → возвращает `"Туристический автобус"`
- `calculate_price()` → `distance_km * 35` (дороже обычного)
- `__str__()` – добавляет информацию о туалете и уровне комфорта

## Валидация данных (отдельный файл `validation.py`)

Все проверки данных вынесены в отдельный модуль для переиспользования и соблюдения принципа единственной ответственности.

| Функция | Что проверяет |
|---------|---------------|
| `validate_route(value)` | Маршрут – непустая строка |
| `validate_capacity(value)` | Вместимость > 0 |
| `validate_passengers(value, capacity)` | Пассажиры ≥ 0 и не превышают вместимость |
| `validate_status(value)` | Статус = `parked` или `on_route` |
| `validate_route_type(value)` | Тип маршрута – непустая строка |
| `validate_comfort_level(value)` | Уровень комфорта от 1 до 5 |

## Коллекция `BusFleet` (файл `collection.py`)

Коллекция была реализована во второй лабораторной работе. Для третьей работы она была дополнена методами фильтрации по типу:

- `get_only_city_buses()` – возвращает новую коллекцию только с городскими автобусами
- `get_only_tourist_buses()` – возвращает новую коллекцию только с туристическими автобусами

Коллекция умеет хранить любые объекты, являющиеся наследниками `Bus`. Это демонстрирует полиморфизм на уровне контейнера.
