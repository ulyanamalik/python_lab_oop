# Лабораторная работа №6: Generics и typing

## Цель работы

Освоить систему аннотаций типов в Python, научиться создавать обобщённые (generic) классы с помощью TypeVar и Generic, понять концепцию структурной типизации через Protocol.

## Описание реализованных типов и контейнеров

### 1. Аннотации типов в классах

Все классы из предыдущих лабораторных (Bus, CityBus, TouristBus) дополнены аннотациями типов. Теперь каждый параметр конструктора, каждый атрибут и возвращаемое значение метода имеют указание типа. Это улучшает читаемость кода и позволяет IDE находить ошибки на этапе написания.

### 2. Generic-коллекция TypedCollection

Создан новый класс `TypedCollection`, который является обобщённой версией коллекции из второй лабораторной работы. Он может хранить объекты любого типа, но этот тип фиксируется при создании коллекции. Например, можно создать коллекцию для автобусов, для строк или для чисел – и тип будет проверяться.

Класс включает все основные методы из ЛР-2: добавление, удаление, получение всех элементов, а также магические методы для получения длины, итерации и доступа по индексу.

### 3. TypeVar без ограничений

Используются два параметра типа:

- `T` – обозначает тип элементов, хранящихся в коллекции
- `R` – обозначает тип результата при преобразовании (для метода map)

### 4. Методы find, filter, map

В коллекцию добавлены три функциональных метода:

- **find** – принимает функцию-условие и возвращает первый подходящий элемент или None
- **filter** – принимает функцию-условие и возвращает список всех подходящих элементов
- **map** – принимает функцию преобразования и возвращает список результатов. Важная особенность: тип результата может отличаться от типа элементов коллекции

### 5. Протоколы (структурная типизация)

Созданы два протокола:

- **Displayable** – требует наличия метода `display`, возвращающего строку
- **Scorable** – требует наличия метода `score`, возвращающего число

Протоколы описывают не «кем является» объект, а «что умеет делать». Это ключевая идея структурной типизации.

### 6. TypeVar с ограничениями

Созданы параметры типа с ограничением (bound):

- `D` – может быть только типом, у которого есть метод `display`
- `S` – может быть только типом, у которого есть метод `score`

Это позволяет создать специализированные коллекции, которые принимают только объекты с определённым поведением.

### 7. Специализированные коллекции

- `DisplayableCollection` – коллекция для объектов, у которых есть метод `display`
- `ScorableCollection` – коллекция для объектов, у которых есть метод `score`

Отбираются все автобусы, находящиеся на маршруте. Выводится список их маршрутов. Демонстрируется, как метод `filter` возвращает новую коллекцию, а не изменяет исходную.

### Сценарий 5: Метод map с разными типами результата
Метод `map` применяется трижды с разными функциями преобразования:
- Первое преобразование извлекает маршрут каждого автобуса – результат список строк
- Второе преобразование извлекает вместимость – результат список чисел
- Третье преобразование создаёт форматированную строку – результат список строк

Это наглядно показывает, зачем нужен второй параметр типа `R`: тип результата может быть другим.
